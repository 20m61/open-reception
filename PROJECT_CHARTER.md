# Project Charter: open-reception

## 1. プロジェクト概要

open-reception は、iPad を受付端末として利用し、来訪者が担当者を安全・確実に呼び出せる無人受付システムである。
タッチ画面、音声認識、音声合成、VRM アバター、Vonage Video API / WebRTC を組み合わせ、受付の一次対応、担当者呼び出し、遠隔応対、受付履歴の記録を行う。

## 2. 背景

従来の受付では、担当者不在、電話番号の不明確さ、来訪者の操作迷い、受付履歴の未記録、属人的な案内が課題になりやすい。
open-reception は、iPad を固定受付端末として設置し、来訪者が迷わず担当者へ到達できる導線を提供する。

## 3. 目的

- 来訪者が 30 秒以内に目的の担当者または部署へ呼び出し開始できる
- 担当者が iPad 受付からの着信・通知を即座に認識できる
- 通話不可時にも代替導線を提示できる
- 音声とアバターにより、無人受付でも心理的に冷たくない体験を提供する
- セキュリティ、プライバシー、運用安定性を初期設計から担保する

## 4. 成功指標

- iPad 実機で主要受付フローが完走できる
- 来訪者が担当者検索、部署選択、呼び出し、待機、失敗時代替導線まで迷わず進める
- 呼び出し成功・失敗・キャンセル・不在時対応が履歴化される
- 主要画面の e2e テスト、ユニットテスト、静的解析、依存関係チェックが CI で実行される
- 受付端末として 1 営業日以上の連続稼働テストに耐える

## 5. スコープ

### In Scope

- iPad 向け受付 UI
- 担当者・部署・呼び出し先の管理
- 来訪者受付フロー
- 音声認識・音声合成
- VRM アバター表示、待機、案内、呼び出し中リアクション
- Vonage を利用した担当者呼び出し、遠隔応対
- 受付履歴、操作ログ、障害ログ
- Semgrep、OWASP ASVS 参照、依存関係監査、Secret scan、各種自動テスト

### Out of Scope / 後続検討

- 物理入館ゲート連携
- 社員証・QR 入館証の発行
- 顔認証
- 高度な予約管理
- 多拠点統合管理
- ネイティブ iPad アプリ化

## 6. 制約

- 初期受付端末は iPad / iPadOS / Safari または PWA を想定する
- マイク・カメラ・自動再生・スリープ・ネットワーク切断など iPadOS の制約を考慮する
- 来訪者の個人情報は最小限にする
- Vonage 認証情報、API Secret、署名鍵はクライアントに置かない
- 音声認識はブラウザ実装差が大きいため、必ずタッチ操作のみで完結できるフォールバックを用意する

## 7. ステークホルダー

- 来訪者
- 呼び出される担当者
- 受付運用管理者
- システム管理者
- セキュリティ/監査担当

## 8. 初期マイルストーン

1. 憲章・要件・仕様・品質方針の整理
2. iPad 受付 UI の MVP 実装
3. 担当者/部署管理と呼び出し先管理
4. Vonage 通話・通知連携
5. VRM アバターと音声案内
6. 受付履歴・監査ログ
7. セキュリティ/テスト自動化
8. 実機 UAT と運用手順整備
