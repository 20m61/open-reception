# UI レビュー（2026-06-27）

`tests/e2e/capture-screens.spec.ts` で取得した各画面のスクリーンショット（`screenshots/`）を
目視レビューした記録。カスタマージャーニー（`docs/customer-journeys.md`）の各タッチポイントと、
今回追加した発行/エンロール/LP の新画面を中心に評価する。

対象（抜粋）: `lp-00-landing` / `admin-00-login` / `admin-05b-device-issued` /
`kiosk-06-enroll-error` / `kiosk-01-idle` / `kiosk-05-confirm` ほか admin/platform 一式。

## 総評

- **ブランドの一貫性は良好**。ダーク基調＋シアンのアクセントが LP・ログイン・受付端末で統一され、
  「会社の顔」としてのトーンが通っている。主 CTA（ログイン／呼び出す／発行）が常にシアンで明快。
- **受付端末はタッチファースト**。待機画面は大きなカード＋アイコン＋多言語切替で、立ったまま
  迷わず選べる。確認画面はステッパー（用件→相手→情報→確認）で現在地が分かる。
- **新規の発行モーダル/エンロール画面の完成度が高い**。QR は十分大きく白背景でスキャンしやすい。
  URL コピー・有効期限・「ここでしか表示できません」警告まで揃っている。

総じて**本番投入できる水準**。以下に、効果の高い順で改善点を挙げる。

## 指摘（深刻度順）

### 🔴 H1. 確認画面でヘルプチップが「最初に戻る」ボタンに重なる（既存・要修正）
`kiosk-05-confirm.png`: 右下のフローティング「お困りですか？」チップが、最下段ナビの
**「最初に戻る」ボタンに被って文字が隠れている**（"最初に戻〜" が欠ける）。来訪者が押したい
導線がチップで潰れる視認/操作性の不具合。
→ ヘルプチップとボトムナビの z-order/レイアウトを分離（チップを一段上げる、またはナビ幅を確保）。

### 🟠 M1. 受付URL発行モーダルが縦中央でなく上寄せで、短い縦解像度で見切れる（今回追加）
`admin-05b-device-issued.png`: QR を含む背の高いモーダルが上端に寄り、タイトルが上辺に接して
いる。縦 ~900px 以下では「閉じる」やタイトルが画面外に出るおそれ。
→ `dialogBox` に `max-height: 90vh` + `overflow:auto` を付与し、背景中央寄せでも内部スクロールで
収める（`src/components/admin/DevicesManager.tsx` の発行モーダル）。

### 🟠 M2. エンロール失敗画面の「再試行」が無効/期限切れ/使用済みでは無意味（今回追加）
`kiosk-06-enroll-error.png`: 「URLが無効か期限切れです」に対し「再試行」を出しているが、同じ
トークンの再 POST は必ず再失敗する。再試行が有効なのは `network` エラー時のみ。
→ 端末エラー（invalid/expired/used/revoked）では「再試行」を出さず「管理画面で再発行」を主導線に。
  通信エラー時のみ再試行を表示する（`src/app/kiosk/enroll/page.tsx`）。

### 🟡 L1. 日本語の語中改行が散見（既存・ポリッシュ）
- 待機カード見出し: 「担当者を呼／ぶ」「部署から選／ぶ」「その他のご／用件」。
- カード説明・LP/エラーの補助文: 「…の方はこち／ら」「くださ／い」。
カード幅に対し見出しが折れて、語の途中で改行され見栄えが粗い。
→ 見出しに `word-break: keep-all; overflow-wrap: anywhere;` 調整、またはカード最小幅/字サイズの
  見直し。`lp-00-landing` の補助文も同様に幅を詰める。

### 🟡 L2. 待機カードのグリッドが最終行で不揃い（既存・ポリッシュ）
`kiosk-01-idle.png`: 2 カラムに 5 枚で、最後の「その他のご用件」が左カラム単独になり右が空く。
→ 5 枚目を全幅にする / 並びを調整して視覚的バランスを取る。

### 🟡 L3. 確認画面の「戻る」系コントロールが多い（既存・情報）
`kiosk-05-confirm.png`: 「修正する」「戻る」「最初に戻る」「キャンセル」と後退系が 4 つ。受付端末の
単純さの観点では整理余地（例: 主導線は「修正する」、ボトムナビは「最初に戻る」に集約）。

### ⚪ I1. 待機画面アバターが静止キャプチャでプレースホルダ表示（情報）
`kiosk-01-idle.png` の「AI」円はスクショ環境で VRM/WebGL が描画されないためのフォールバック。
実機描画は別途確認済み（VRM #31）。回帰ではない。

## 良かった点（維持したい）

- 発行モーダルの QR サイズ・コントラスト・コピー/期限/警告の三点セット（`admin-05b`）。
- ログイン画面のパスワード表示トグル（目アイコン）と単一 CTA の明快さ（`admin-00`）。
- 確認画面のステッパーと要約（入力情報のみ表示、PII は完了後に残さない方針は e2e で別途担保）。
- 待機画面の多言語切替を一等地に配置（`kiosk-01`）。

## フォローアップ候補（別 issue 推奨）

- H1（ヘルプチップ重なり）: 受付端末の最重要導線の視認性に関わるため最優先。
- M1/M2: 今回追加分。本 PR スコープで小修正可能（任意）。
- L1–L3: デザインシステムのカード/タイポ調整としてまとめて対応。
