## 並列トラック（初期ウェーブ）

同時に走らせてよい独立トラック。1 ウェーブ = 最大 2〜3 本。

### Wave 1（相互独立・並行）

| トラック | Issue | worktree ブランチ案 | 触る主な領域 |
| --- | --- | --- | --- |
| A | #105 | `feat/compliance-guard` | `docs/`, 依存チェック手順 |
| B | #80 | `feat/multitenant-base` | `src/domain`, `src/lib`, 永続化 |
| C | #79 | `feat/presence-detection` | `src/components/kiosk`, `src/lib` |

> B と C は `src/lib` に触れる可能性があるため、着手時にサブディレクトリ重複を確認し、
> 重なるなら直列化する。A は主に docs で衝突しにくい。

- `#85`（管理画面基盤）着手。`#80` マージ後。

## 進め方メモ

- 各トラックは独立 worktree（または `isolation: "worktree"` のサブエージェント）で実装。
- マージは 1 本ずつ、ゲート green を示してユーザー確認 → squash + `--delete-branch`。
- 後続トラックはマージ後 main を `git pull --ff-only` で取り込んでから整合確認。
- 状態は本ファイルのチェック/表で更新していく。
