# UI レビュー（2026-07-11）

`tests/e2e/capture-screens.spec.ts` + 補足キャプチャ（呼び出し結果系・退館・サイネージ・
英語ロケール・横置き・admin 狭幅 390px）で全 42 画面を取得し目視レビューした記録。
併せて `globals.css` / `components/admin/ui/tokens.ts` / 主要コンポーネントの静的レビューを実施。
前回（`docs/ui-review-2026-06-27.md`）の対応後リグレッションと、当時見送った L3 の再評価を含む。

## 総評

ブランドトーン・タッチファースト設計・ステッパー・監査ログのフィルタ UI など基礎品質は高い。
一方で「**文言の重複/競合**」「**後退系ボタンの氾濫（同一ボタンの二重表示）**」「**多言語の漏れ**」
「**デザイントークンの真実源分裂**」という、体験の芯に関わる系統的な課題を確認した。
指摘はすべて issue 化済み（#324〜#331）。

## 指摘（深刻度順）

### 🔴 H1. 同一画面に「最初に戻る」が 2 個（→ #325）
`kiosk-09-fallback` / `kiosk-10-result-connected`: コンテンツ内ボタンと逃げ道バーで同一機能が
二重表示。`kiosk-04-visitor-info` はフォーム「戻る」+ バー「戻る」で後退系 4 種。
2026-06-27 L3（見送り）が実害化しており、後退語彙の定義と表示位置の一本化が必要。

### 🔴 H2. English モードに日本語導線が残存（→ #327）
`kiosk-15-idle-en`: 「退館チェックアウト」ボタンが日本語のまま。`/kiosk/checkout` は全文言
日本語ハードコード・locale 非連動。ロケール網羅の機械検証が無いことが根因。

### 🟠 M1. 待機画面の三重案内と用件の二重質問（→ #324）
`kiosk-01-idle`: 見出し「ご用件をお選びください」・リード「タッチして開始」・アバター字幕
「タッチして始めてください」が競合。用件系カードを選んでも step1 で再度「ご用件をお選びください」
（面会/納品/…）を質問。connected 画面の主 CTA「受付を終了する」も役割曖昧。

### 🟠 M2. 退館の自己特定が stayId 頼み（→ #328）
`kiosk-13-checkout`: 受付番号プレースホルダが `stay-...`（内部 ID）。在館一覧も stayId +
入館時刻のみで自分の行を判別できず、誤退館の確認も無い。ページ全体が生 HEX の独自スタイルで
デザイン系統からも外れている。

### 🟠 M3. トークンの真実源が分裂（→ #329）
`tokens.ts` radius `8/12/16` vs CSS `--radius 10/14/18`、border `0.1/0.2` vs `0.08/0.16` の乖離。
`style={{` が components 配下に 700+ 箇所、#92 inc1 が宣言した「次増分での移行」が未完。
生 HEX 持ち込みを防ぐガードなし。

### 🟠 M4. 管理画面に内部注記が露出・一覧の運用性不足（→ #330）
`admin-01-dashboard`: 「実データは #82/#90 で接続予定（design 注記）」等が本番 UI に表示。
受付履歴は期間フィルタ/ページング/CSV なし（監査ログには有り、非対称）。結果列に raw code
（call_failed 等）併記。サイドバー語中改行（ダッシュボー/ド 等）。390px でテーブルが縦書き化
（`admin-narrow-03`、#94 AC 未達）。保存フィードバックが Manager ごとに独自実装。

### 🟡 L1. 画面見出しの語中改行・横置きカードの文字あふれ（→ #326）
`kiosk-03`「…お選びくださ／い」、`kiosk-04`「…入力してくださ／い」（screen__title は前回 L1
対応の対象外だった）。横置き 4 列で「担当者を呼ぶ」端接触・「QR／で受付」語中折れ
（`kiosk-16`）。結果画面の死空間・サイネージ未設定時の黒画面（`kiosk-14`）・LP タグライン折返し。

### 🟡 L2. ホーム画面設置・タブ体験の資材が無い（→ #331）
favicon/アイコン/manifest が皆無。全ページタイトル "open-reception" 固定。standalone 表示・
スプラッシュ未定義で PWA 運用（charter §6）の設置体験が未整備。

## 良かった点（維持したい）

- 監査ログのフィルタ群（期間/種別/主体/キーワード + 件数表示）は一覧 UI の模範。受付履歴にも展開したい。
- ステッパー・確認必須の不変条件・無操作カウントダウン等、UX 契約（#120/#121）の実装は堅牢。
- 担当者一覧の在席/有効の色語彙、不在者の選択不可 + 代替案内（`kiosk-03` の大野）は明快。
- reduced-motion / focus-visible / WCAG 1.4.4（ピンチズーム許可）などの下回りが誠実。
