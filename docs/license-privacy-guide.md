# ライセンス・権利・プライバシー確認ガイド (issue #105)

open-reception に取り込む外部要素（OSS 依存・QR / TTS / STT / LLM / 翻訳 API・VRM /
モーション / 画像 / 動画 / BGM 素材・フォント / アイコン・UI ライブラリ）について、
ライセンス・利用規約・商標・個人情報の確認手順を定める。機能拡張のたびに毎回使える
チェックリストと運用を提供することが目的。

**前提方針**

- 機能思想は参考にできるが、**実装・UI・文言・画像・コードはコピーしない**。
- OSS は**ライセンスが明示されているもの**を優先する。不明な素材・ライブラリは採用しない。
- **顔認証・録画・録音・生体情報は初期スコープ外**とする。
- 個人情報は**必要最小限・保存期間明示・監査ログ最小化**を原則にする。
- **GitHub Actions は使用しない方針**のため、依存ライセンスチェックは
  ローカル / スクリプトで完結させる（`docs/loop-workflow.md` 参照）。

このガイドは法務レビューの代替ではない。商用利用や再配布に疑義がある場合は採用を保留し、
判断を記録する（[判断ログ](#判断ログ運用)）。

---

## 0. クイックチェック（採用前に必ず通す）

新しい外部要素（依存・API・素材）を取り込む前に、最低限これを満たすこと。

- [ ] ライセンス / 利用規約が**明示**されている（不明なものは採用しない）。
- [ ] 用途（商用 / 無人受付端末での常時表示 / SaaS 配信）が許可されている。
- [ ] 改変・再配布・帰属表示の条件を満たせる（NOTICE / クレジットの置き場所を決めた）。
- [ ] 競合 SaaS の UI / 文言 / 画像 / アイコン / コードを流用していない。
- [ ] 個人情報・音声・映像・生体情報を新たに収集しない、または収集が最小限で根拠がある。
- [ ] 秘密情報（API キー / private key）をフロント bundle・管理画面に露出しない。
- [ ] [判断ログ](#判断ログ運用)に記録した。

いずれかが NO の場合は採用を保留し、代替を探すか判断を記録する。

## 1. OSS 依存追加時のチェックリスト

外部 npm パッケージ（直接・推移を問わず主要なもの）を追加するときに確認する。

### 1.1 ライセンス確認

- [ ] パッケージの **SPDX ライセンス識別子**を確認した（`package.json` の `license`
      フィールド、または同梱 `LICENSE` ファイル）。
- [ ] ライセンス種別が**許容リスト**に入る（[許容ライセンス](#13-許容ライセンスの目安)）。
- [ ] **copyleft（GPL / AGPL / LGPL 等）** でないことを確認した。やむを得ず使う場合は
      配信形態（SaaS / バイナリ）への影響を判断ログに記録する。
- [ ] 推移的依存（transitive）に問題ライセンスが含まれないことを確認した
      （`npx license-checker` 等の候補は [3 章](#3-ローカル依存ライセンスチェック手順) 参照）。
- [ ] **デュアルライセンス**や**カスタムライセンス**の場合、適用条件を読んだ。
- [ ] ライセンス不明・`UNLICENSED`・空のパッケージは**採用しない**。

### 1.2 帰属・NOTICE の扱い

- [ ] 帰属表示（attribution）が必要なライセンス（MIT / BSD / Apache-2.0 等）について、
      クレジットの掲載方法を決めた（後述の [NOTICE 運用](#14-notice--クレジットの運用)）。
- [ ] Apache-2.0 等で `NOTICE` ファイルが同梱される場合、その内容を NOTICE に転記する。

### 1.3 許容ライセンスの目安

| 区分 | 例 | 扱い |
| --- | --- | --- |
| 許容（permissive） | MIT, ISC, BSD-2/3-Clause, Apache-2.0, 0BSD, Unlicense | 帰属条件を満たせば採用可 |
| 条件付き（要判断） | MPL-2.0, LGPL（動的リンク前提）, CC-BY（素材） | 配信形態・改変範囲を判断ログに記録して採用可否を決める |
| 原則回避 | GPL-2.0/3.0, AGPL-3.0, SSPL, CC-BY-NC, CC-BY-SA, 商用不可・再配布不可 | 採用しない（代替を探す）。例外は明確な根拠と記録が要る |
| 採用不可 | ライセンス不明, `UNLICENSED`, カスタムで商用禁止 | 採用しない |

> AGPL / SSPL は SaaS 配信（OpenNext on AWS）でソース開示義務が及び得るため原則回避。
> CC-BY-NC（非商用）は受付システムが商用利用に当たり得るため素材でも回避。

### 1.4 NOTICE / クレジットの運用

- 帰属が必要な OSS・素材は、リポジトリ直下に **`THIRD_PARTY_NOTICES.md`**（または
  `NOTICE`）を設けて集約する（このガイドでは置き場所のみ定義し、生成は依存追加時に行う）。
- ランタイムでクレジット表示が必要な素材（CC-BY のフォント / 画像 / BGM 等）は、
  管理画面または「このアプリについて」相当のページに表示する場所を決める。
- `THIRD_PARTY_NOTICES.md` は `npx license-checker` 等の出力を起点に手動で整える
  （[3 章](#3-ローカル依存ライセンスチェック手順)）。

## 2. 確認対象別の観点

### 2.1 QR 生成 / 読み取りライブラリ

- [ ] 生成（encode）・読み取り（decode）ライブラリの SPDX を確認（多くは MIT / Apache-2.0）。
- [ ] **特許**条項に注意（QR コードの基本仕様自体はロイヤリティフリーだが、装飾 QR・
      フレーム QR・独自拡張は別途権利があり得る）。
- [ ] 読み取りはカメラ映像を扱うため、[2.7 個人情報・音声・映像・生体情報](#27-個人情報音声映像生体情報の初期スコープ)
      の原則（映像はローカル処理・保存しない）に従う。
- [ ] QR に**個人情報や長期有効トークンを直接埋め込まない**（短命トークン参照に留める）。

### 2.2 通話 / WebRTC SDK

- [ ] SDK のライセンス・利用規約・商用プラン条件を確認（Vonage 等の SaaS は別途規約）。
- [ ] secret / private key を**フロント bundle・管理画面に露出しない**
      （`docs/security-checklist.md` の V6、`client-secret-guard` テスト参照）。
- [ ] 通話の録画・録音は初期スコープ外。SDK の録画機能を**有効化しない**。
- [ ] 通話メタデータ（誰がいつ）を監査ログに残す場合も PII は最小化する
      （`docs/audit-logging.md` と整合）。

### 2.3 TTS / STT / LLM / 翻訳 API（外部 AI サービス）

外部 AI / クラウド API は OSS とは別に**利用規約・データ取り扱い**を確認する。

- [ ] **入力データの学習利用**の有無と、オプトアウト可否を確認した。
- [ ] **データの保存場所・保存期間・リージョン**（データレジデンシー）を確認した。
- [ ] **出力物の権利・利用範囲**（商用利用可否、生成音声 / 翻訳文の再利用）を確認した。
- [ ] 送信するデータに**来訪者の PII・音声・映像を含めない、または最小化**する。
- [ ] STT に渡す音声は**保存しない / 即時破棄**を前提に設計する（録音はスコープ外）。
- [ ] API キーは server-only 環境変数で扱い、`NEXT_PUBLIC_` を付けない。
- [ ] レート制限・障害時のフォールバック方針を決めた（無音 / 既定文言など）。
- [ ] 利用規約の**ブランド表記要件**（"Powered by" 等）があれば順守する。

### 2.4 VRM モデル / アバター

- [ ] VRM の **`VRMC_vrm` メタデータ内のライセンス情報**（許諾項目）を確認した
      （商用利用 / 改変 / 再配布 / クレジット表示の各可否）。
- [ ] アバターの**人格・名前・声**に関する制約（なりすまし禁止等）を確認した。
- [ ] 自作・委託・購入のいずれでも、**利用許諾の証跡**を判断ログに残す。
- [ ] 不明な出所のモデルは採用しない。

### 2.5 モーションデータ

- [ ] モーション（BVH / VMD / アニメーション）の商用利用・改変・再配布可否を確認した。
- [ ] VRM とモーションの**ライセンスが別**であることに注意（それぞれ確認）。
- [ ] モーションキャプチャ元の権利（実在人物の動き等）に問題がないことを確認した。

### 2.6 背景画像 / 動画 / BGM / フォント / アイコン

- [ ] 各素材の**商用利用・改変・再配布可否**と**クレジット要否**を記録した。
- [ ] ストックサイト素材はライセンス種別（CC0 / 標準 / 拡張）と利用範囲を確認した。
- [ ] フォントは**埋め込み・Web 配信（@font-face）・バンドル**の各可否を確認した
      （SIL OFL 等）。
- [ ] アイコンセットのライセンスと帰属要件を確認した（CC-BY は帰属必須）。
- [ ] BGM の**ループ再生・公共空間での再生**に制約がないか確認した。
- [ ] 帰属が必要なものは [NOTICE 運用](#14-notice--クレジットの運用)に登録する。

### 2.7 個人情報・音声・映像・生体情報の初期スコープ

**初期スコープ外（本番導入しない）**

- 顔認証 / 顔特徴量の抽出・保存
- 通話 / 受付の録画・録音
- 指紋・声紋・虹彩などの**生体情報**の収集・保存

**収集する場合の原則（必要最小限）**

- [ ] **必要最小限**: 受付に不可欠な項目のみ収集する（氏名・訪問先・用件など）。
- [ ] **保存期間を明示**: 各データの保存期間を決め、超過分は破棄する設計にする。
- [ ] **監査ログ最小化**: 監査ログに来訪者 PII を残さない（`docs/audit-logging.md`、
      `docs/security-checklist.md` の V7/V8 と整合）。
- [ ] **画面からの消去**: 受付完了 / キャンセル後に画面から個人情報を消す。
- [ ] カメラ映像（QR 読み取り等）は**ローカル処理**を基本とし、サーバへ送らない / 保存しない。
- [ ] 音声（STT 等）は即時処理・即時破棄を基本とし、録音保存しない。

### 2.8 商標・ブランド名・ロゴ・競合 SaaS

- [ ] 製品名 / ロゴ / 商標を**無断で UI・ドキュメント・素材に使わない**。
- [ ] 「〇〇互換」「〇〇風」等の表現で第三者ブランドに**便乗しない**。
- [ ] 競合・類似 SaaS の **UI / 文言 / 画面構成 / 画像 / アイコン / コードを流用しない**
      （機能思想の参考は可、表現のコピーは不可）。
- [ ] 参考にした場合でも、配色・レイアウト・コピーは**自前で起こす**。
- [ ] 第三者 API のブランド表記要件（ロゴガイドライン等）がある場合のみ、その規定に従う。

## 3. ローカル依存ライセンスチェック手順

**GitHub Actions を使わない方針**のため、依存ライセンスの確認はローカル / スクリプトで
完結させる。現状リポジトリには専用の license スクリプトは無いため、以下は**導入候補の
チェックリスト**として扱い、勝手に依存追加・スクリプト追加はしない（採用時に別 Issue / PR）。

### 3.1 いま使える手順（依存追加なし）

- [ ] `npm run audit:deps`（= `npm audit --omit=dev`）で本番依存の脆弱性を確認する
      （既存スクリプト、`docs/security-checklist.md` と共通）。
- [ ] 個別パッケージのライセンスは `npm view <pkg> license` で確認する。
- [ ] 一時的に全依存のライセンスを俯瞰したいときは、**devDependency に残さず**
      `npx license-checker --summary`（または `--production`）を都度実行する。

### 3.2 導入を検討する候補（採用は別途判断）

| 候補 | 用途 | 導入時の注意 |
| --- | --- | --- |
| `license-checker` / `license-checker-rseidelsohn` | 依存ライセンス一覧・NOTICE 生成 | devDependency 追加が要る。`--onlyAllow` で許容リストを強制可 |
| `npm run audit:deps`（既存） | 脆弱性監査 | 追加不要。ライセンス自体は見ない |
| `oss-review-toolkit` 等 | 大規模なライセンス / コンプラ走査 | 重量級。現段階では過剰の可能性 |

導入する場合の方針:

- `npm run verify` / `scripts/quality-gate.sh` に**ライセンスチェックを組み込む**ことを
  検討する（例: `npm run audit:licenses` を新設し `--audit` 系に束ねる）。未導入ツールは
  既存ゲート同様 **SKIP 表示**にし、`--strict` で FAIL 扱いにできる形を踏襲する。
- 許容ライセンスは [1.3 許容ライセンスの目安](#13-許容ライセンスの目安) を `--onlyAllow`
  相当のリストとして反映する。
- 出力を起点に `THIRD_PARTY_NOTICES.md` を更新する運用にする。

> 受け入れ条件「CI または `npm run verify` に組み込める依存ライセンスチェック候補が
> 整理されている」を満たすための整理であり、本 PR では依存・スクリプトは追加しない。

## 判断ログ運用

採用・保留・回避の判断は**証跡を残す**。軽量に運用するため、以下のいずれかに記録する。

- 採用を決めた PR / Issue 本文に、対象・ライセンス（SPDX）・用途・判断・根拠を書く。
- 帰属が必要なものは `THIRD_PARTY_NOTICES.md`（OSS）または「このアプリについて」相当の
  クレジット欄（ランタイム素材）に反映する。

記録テンプレート:

```
- 対象: <パッケージ / API / 素材名・バージョン / URL>
- ライセンス: <SPDX 識別子 / 利用規約 URL>
- 用途: <どこで何に使うか>
- 商用利用: 可 / 不可 / 要条件（条件: ...）
- 改変 / 再配布: 可 / 不可 / 要条件
- 帰属表示: 要 / 不要（掲載先: ...）
- 個人情報 / 音声 / 映像: 扱う / 扱わない（扱う場合: 最小化・保存期間・破棄方針）
- 判断: 採用 / 保留 / 回避（根拠: ...）
```
