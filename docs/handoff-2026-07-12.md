## 5. 再開の手引き（運用メモ）

- **ブランチ**: 開発は `claude/loop-track-implementation-18syan`。**マージ済みのため follow-up は
  main から作り直す**（`git fetch origin main && git checkout -B <branch> origin/main`）。
- **マージ運用**: PR は squash + `Closes #N`。本セッションは 1 本ずつ（またはトラック束）で
  ゲート green を確認して自律マージした（additive/非破壊のため重大変更に非該当）。**重大変更
  （スキーマ/公開API/PII/本番デプロイ/破壊的）はユーザー確認**。
- **サブエージェント運用**: `Agent` の `subagent_type:"loop-track"` + `isolation:"worktree"`。
  各エージェントは AC を実コードへマッピングし**未充足のみ TDD**。push/PR/merge はさせず、
  オーケストレータが cherry-pick → `./scripts/quality-gate.sh --pr` green → push/PR/squash。
- **トポロジ注意**: squash マージで main が枝分かれするため、次トラック着手前に
  `git merge origin/main` で main を ancestor に取り込む（force-push はサンドボックスで
  ブロックされる。**force-push/remote 削除は不可**、非 fast-forward を作らない運用で回避した）。
- **ゲート**: `./scripts/quality-gate.sh --pr`（typecheck/lint/unit/build）を各統合で緑に。
  Chromium は `/opt/pw-browsers` で利用可（e2e は回せる）。lighthouse は Chrome 不在で不可。
  `no-raw-color-literals` eslint と i18n locale 網羅テスト（`ja-simple` は部分許可）に注意。
- **worktree 掃除**: `git worktree list` の全エントリを撤去してから次へ。
