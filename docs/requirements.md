# Requirements: open-reception

## 1. プロダクト要件

open-reception は、iPad を受付端末として設置し、来訪者が担当者または部署を選択して呼び出せる無人受付システムである。
受付体験はタッチ操作を主導線とし、音声認識・音声合成・VRM アバターを補助導線として利用する。

## 2. 利用者種別

| 利用者 | 目的 |
| --- | --- |
| 来訪者 | 担当者を呼び出す、受付状況を理解する |
| 担当者 | 来訪通知を受け取り、応対可否を判断し、必要に応じて通話する |
| 受付運用管理者 | 担当者、部署、呼び出し先、表示文言、端末状態を管理する |
| システム管理者 | セキュリティ、監査ログ、障害対応、設定管理を行う |

## 3. MVP ユースケース

### UC-01: 担当者を呼び出す

1. 来訪者が iPad の受付開始ボタンを押す
2. 目的を選択する
3. 担当者名または部署を検索/選択する
4. 来訪者名、会社名、要件を入力する
5. 呼び出しボタンを押す
6. 担当者に通知または Vonage 通話を開始する
7. iPad に呼び出し中、応答、失敗、代替案内を表示する

### UC-02: 部署を呼び出す

1. 来訪者が部署を選択する
2. 部署に紐づく代表呼び出し先へ通知する
3. 複数担当者がいる場合、優先順位または同報で呼び出す

### UC-03: 担当者が応答できない

1. 一定時間応答がない
2. 代替担当者、受付電話、メッセージ入力、再呼び出しを提示する
3. 履歴に不在・未応答として記録する

### UC-04: 音声とアバターによる案内

1. VRM アバターが待機状態で挨拶する
2. 操作ごとに短い音声案内を行う
3. 音声認識で担当者名や部署名を拾える場合は候補を提示する
4. 音声が使えない場合も、タッチ操作のみで完結できる

### UC-05: 端末運用

1. iPad は受付専用画面として固定する
2. 通信断、マイク権限なし、カメラ権限なし、音声合成不可、Vonage 接続失敗を検知する
3. 端末状態を管理画面またはログで確認できる

## 4. 機能要件

### 4.1 iPad 受付 UI

- タッチ操作だけで全機能を完結できる
- 受付開始、担当者/部署選択、検索、入力、確認、呼び出し、完了画面を提供する
- 受付中の状態を大きく表示する
- 誤操作を戻せる
- 受付完了後は一定時間で待機画面に戻る
- 画面サイズ、横向き/縦向き、Safari/PWA 表示差を考慮する

### 4.2 担当者・部署管理

- 担当者名、部署、呼び出し先、通知可否、優先順位を管理できる
- 担当者の一時不在、受付停止、代替担当者を設定できる
- 検索用のよみがな、別名、英字表記を持てる

### 4.3 呼び出し

- Vonage によるリアルタイム通話を開始できる
- 通話前に担当者へ着信/通知できる
- 応答なし・拒否・通信失敗・タイムアウトを区別できる
- 代替導線を表示できる

### 4.4 音声認識・音声合成

- 音声認識は補助機能とし、必須導線にしない
- 音声合成で挨拶、操作案内、待機案内、エラー案内を再生する
- 音声再生がブロックされる場合は、ユーザー操作後に再生を開始する
- マイク権限がない場合はタッチ UI にフォールバックする

### 4.5 VRM アバター

- 待機、挨拶、考え中、呼び出し中、成功、失敗の状態を表現する
- 音声合成に合わせて口パクまたは簡易リップシンクする
- iPad の GPU/メモリ負荷を考慮し、軽量モデル・解像度制御・FPS 制御を行う

### 4.6 履歴・ログ

- 受付日時、呼び出し対象、結果、所要時間、失敗理由を記録する
- 個人情報は最小限にし、保存期間を設定できる
- エラー、通信状態、端末状態、権限状態を診断できる

## 5. 非機能要件

### 5.1 セキュリティ

- Vonage のセッション/トークン生成はサーバー側で行う
- API Secret や秘密鍵をフロントエンドに置かない
- 認証・認可を管理画面と端末画面で分離する
- CSRF/XSS/認可不備/ログの個人情報漏洩に対策する
- 監査ログを残す

### 5.2 プライバシー

- 来訪者の入力情報は必要最小限にする
- 音声データを保存する場合は明示的な理由と保存期間を定義する
- 通話録画・録音は初期スコープ外とし、導入時は明示同意を必要とする

### 5.3 可用性

- 通信断でも受付不可表示または代替連絡先を提示する
- iPad スリープ、アプリ終了、リロード、権限失効から復帰できる
- 障害時に手動受付へ切り替えられる

### 5.4 性能

- 初期表示 3 秒以内を目標とする
- 主要操作のレスポンスは 1 秒以内を目標とする
- VRM 表示中でも iPad で操作遅延が発生しない

### 5.5 アクセシビリティ

- 大きなボタン、十分なコントラスト、読みやすい文字サイズを採用する
- 音声が聞こえない/使えない場合でも画面で完結できる
- 日本語を初期対応し、英語・韓国語など多言語対応を後続検討する

## 6. 受け入れ条件の例

- iPad 実機で担当者呼び出しフローが完走できる
- 音声認識が使えなくてもタッチ操作で完走できる
- Vonage 接続失敗時に代替導線が表示される
- 受付履歴に結果と失敗理由が記録される
- CI で lint、unit、e2e、Semgrep、依存関係監査が実行される
