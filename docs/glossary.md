# 用語集 (issue #1)

open-reception で使う主要用語を定義する。

| 用語 | 定義 |
| --- | --- |
| 受付端末 / kiosk | 来訪者が操作する iPad 受付端末。`/kiosk` で表示する。 |
| 受付セッション (ReceptionSession) | 1 回の受付フローの状態・入力・結果を保持する単位。状態遷移モデルで制御する。 |
| 状態遷移モデル | 待機→目的→担当者/部署→入力→確認→呼び出し→結果→完了 を表す受付フローの状態機械（不正遷移を禁止）。 |
| 目的 (purpose) | 来訪の用件（面会/納品/打ち合わせ/その他）。 |
| 呼び出し先 (target) | 呼び出す担当者または部署。 |
| ディレクトリ | 部署・担当者の集合。管理画面と受付端末・呼び出し adapter が共有する単一の情報源。 |
| CallAdapter | 呼び出しの抽象境界。MockCallAdapter（成功/未応答/失敗/タイムアウト）と本番 Vonage 実装を差し替える。 |
| 代替導線 (fallback) | 未応答/失敗時に代表窓口などへ案内する導線。 |
| 受付履歴 (ReceptionLog) | 呼び出し結果・所要時間・代替導線利用などの運用ログ。来訪者の PII は保持しない。 |
| 監査ログ (AuditLog) | 受付ライフサイクルと管理操作（部署/担当者/端末/セキュリティ/音声/アセット変更）の証跡。 |
| kiosk セッション | 受付端末の長期許可を表す署名付き cookie（role=kiosk）。管理 API には使えない。 |
| 管理セッション | 管理者の認証を表す署名付き cookie（role=admin）。`/admin/*`・`/api/admin/*` で必須。 |
| 端末レジストリ / 失効 | 受付端末を識別・登録し、失効（無効化）できる仕組み。失効端末は受付を開始できない。 |
| heartbeat | 受付端末が定期的に端末有効性・許可状態・緊急停止を確認する仕組み。 |
| 緊急停止モード | 全受付端末の受付を一括停止する管理操作。 |
| アセット (Asset) | 背景画像・VRM モデル・モーション・fallback 画像。種別ごとに管理する。 |
| アクティブセット | 受付画面に適用するアセットの選択（種別ごとに 1 つ）。 |
| 音声設定 (VoiceSettings) | 音声合成(TTS)/音声認識(STT)・案内文言・話速・音量・言語。既定はテキスト主導（音声無効）。 |
| 案内文言 (guidance) | 各画面に表示する案内テキスト。音声不可でもテキストで成立する。 |
| StorageAdapter | アセットの保存・参照 URL 取得の抽象境界（本番 S3 等）。 |
